The Computer's Notebook

Memory, the stack, and the picture of an overflow
The Brief three panels // gated

Three panels, in order. Each one teaches one idea about how a computer remembers things, and each one unlocks the next.

Panel 1 — the memory grid. Click any cell to write a byte. Panel 2 — the stack. Push and pop function frames; match a target stack to earn the first half of the flag. Panel 3 — the overflow demo. Type into a 4-byte slot. See what happens past the 4th character. Find the second half of the flag.

Submit the full flag at the bottom when you have both halves.

Panel 1 — Memory Grid (256 bytes) click any cell to write a byte

Below is 256 bytes of memory laid out in a 16×16 grid. Each cell is one byte — its address is shown by its row and column. The cell at row 0x30 column 0x4 has address 0x34.

Click any cell, type a two-digit hex value (00–FF), and press Enter. The cell updates. Memory is just numbered desks holding numbered slips — you are the CPU now.

Write to any 3 distinct cells to confirm you've got the picture. Panel 2 will unlock.
Panel 2 — The Stack match the target stack

Each call pushes a fresh plate (a stack frame) onto the top of the stack. Each plate carries a fake return address, which is what the CPU would jump to when the function returns. return pops the top plate.

Your job: build the target stack below. The target shows the stack from bottom to top — the leftmost plate is the bottom (oldest), the rightmost is the top (newest, most recently called). main is already on the stack for you.

Panel 3 — The Overflow type into the 4-byte slot

Two slots side by side: a 4-byte input slot and the 4-byte return-address slot. They sit next to each other in memory. The return address starts as 0x00DEAD00.

Type into the input below. The first 4 characters fill the input slot byte-by-byte. ASCII values appear in hex inside each byte cell. The 5th and beyond have nowhere to go inside the input slot — so they spill into the next slot. That next slot is the return address.

This isn't a hack — that's just where the slots happen to be next to each other in memory. Try it.
