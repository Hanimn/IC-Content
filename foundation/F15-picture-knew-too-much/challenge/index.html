The Picture That Knew Too Much

A photo arrived from a stranger. Read its hidden notebook. Then look below the pixels and find what was woven inside.

BRIEFING
Mission Briefing

A photograph was forwarded to the Agency by an anonymous source. It looks unremarkable — a simple landscape: sky, sun, ground. But every photo carries more than what your eye sees.

Your task: recover both flag fragments hidden in the image and combine them.

  • Part 1. Read the EXIF metadata block. The photo's notebook contains the first half of the flag.
  • Part 2. Extract the LSB-encoded message from the second image. The lowest red-channel bit of every pixel carries a hidden ASCII string — the second half of the flag.

PART 1
Read the EXIF Metadata

Below is the photograph that arrived. To you it is just a picture. To the file format, it is a picture plus a small notebook of fields describing how, when, and where it was taken. Click the button to read the notebook.

PART 2
Extract the LSB Message

A second image was attached. It is small (32 × 16 pixels — shown zoomed below) and looks like a coloured gradient. Inside it, the lowest bit of every red channel carries a hidden ASCII message.

The algorithm: read each pixel's R value, take only its last bit (R & 1), collect bits in scan order, group into bytes (8 bits each, MSB first), decode as ASCII. Stop when you hit a zero byte.

FILE: gradient.png  |  32 × 16  |  512 pixels = 512 hidden bits = 64 bytes capacity

Step 1: R-channel LSBs (bits in scan order, first 240)
Step 2: Grouped into bytes (8 bits → hex)
Step 3: Decoded as ASCII (until null byte)

GUIDE
How to Solve
Part 1. Click Read Metadata. Look at every line of the EXIF dump. One field will contain a fragment of the flag in plain text. Read carefully — the Comment field is a classic hiding spot.
Part 2. Click Extract Hidden Message. The page reads each pixel's red value with getImageData, masks off the lowest bit, groups every 8 bits into a byte, and decodes ASCII. The output appears in green.
Part 3. Concatenate fragment 1 and fragment 2 (no extra spaces) and submit.
REF
LSB Quick Reference
SymbolMeaning
LSBLeast Significant Bit (the lowest bit, value 1)
R, G, BRed, Green, Blue — each 0–255
v & 1Mask off everything but the lowest bit
8 bitsGroup size for one ASCII character
0x00Null byte — common end-of-string marker

A 32 × 16 image has 512 pixels. With one bit per pixel (R-channel only), that's 64 hidden bytes — enough for a short message.
